# 2.1 — Scaffold `@modeler/sql`: vendor + generate lexers/parsers

**Repo:** modeler. **Effort:** ~1 day. **Pre-flight:** Phase 0 GO; Phase 1
merged. Read SPIKE S0.1 (generation, Postgres base-class injection) and
[`PINNED.md`](PINNED.md) (grammars-v4 `@923a1a9`, `antlr-ng@1.0.10`, command).
This stage reproduces the spike's generation as **production** code.

---

- [ ] **2.1.1 — Create the package.** `packages/sql` → `@modeler/sql`, `private`
      not set (it ships), extends `tsconfig.base.json`, ESM, deps `antlr4ng`.
      Declare **subpath exports** (the E11 bundle split):
      ```jsonc
      "exports": {
        ".":        { "import": "./dist/index.js", "types": "./dist/index.d.ts" },
        "./lexers": { "import": "./dist/lexers.js", "types": "./dist/lexers.d.ts" }
      }
      ```
      `./lexers` re-exports only the dialect lexers + `maskPlaceholders` (browser);
      `.` adds parsers + adapters (desktop).

- [ ] **2.1.2 — Vendor the grammars (pinned).** Add `vendor/tsql/` and
      `vendor/postgresql/` from grammars-v4 `@923a1a9` (the lexer/parser `.g4` +
      the `Antlr4ng/` base classes + `transformGrammar.py`). Commit `vendor/` as-is
      (no edits — lazy patching, DESIGN §12.7). Record the SHA in
      `packages/sql/vendor/PINNED.md`.

- [ ] **2.1.3 — Port the generation script.** `packages/sql/scripts/generate.sh`
      (from the spike): for each dialect, copy `.g4` to a throwaway dir, for
      Postgres inject the `@header` import of `PostgreSQLParserBase`/`LexerBase`
      (replicating `transformGrammar.py`), run
      `npx antlr-ng -Dlanguage=TypeScript -o src/generated/<dialect> --lib <dir> -- <Lexer.g4> <Parser.g4>`,
      and copy the `Antlr4ng/*.ts` base classes beside the output. T-SQL needs no
      injection. Add `"prebuild": "bash scripts/generate.sh"` (mirrors
      `@modeler/parser`). `src/generated/**` is **gitignored** (regenerated), like
      the TTR parser.

- [ ] **2.1.4 — ESLint exemption for generated code.** Add
      `packages/sql/src/generated/**` to the ESLint ignore (the repo already
      exempts `generated/**` for `any`). Confirm `tsc --noEmit` is clean over
      `generated/**` (SPIKE S0.1 got exit 0).

- [ ] **2.1.5 — Generation smoke test.** Port the spike's smoke test:
      `src/__tests__/generate-smoke.test.ts` — for tsql + postgres, lex `SELECT 1`
      → non-empty token stream; parse `SELECT 1;` → non-empty tree; assert
      `select`/`SELECT`/`SeLeCt` all yield the keyword token (case-insensitivity).
      ```bash
      pnpm --filter @modeler/sql build && pnpm --filter @modeler/sql test
      ```

- [ ] **2.1.6 — Wire into the workspace; commit.** Confirm `pnpm -r build` builds
      `@modeler/sql` (prebuild runs generation). Expect the two benign Postgres
      empty-string lexer warnings (SPIKE S0.1) — document them in a
      `packages/sql/README.md` "known warnings" note so they're not re-triaged.
      Commit: `Section embedded-sql: @modeler/sql scaffold + grammar generation`.
