# 4.3 — Find-references (TTR `db` symbol → SQL usages)

**Repo:** modeler. **Effort:** ~1 day. **Package:** `@modeler/lsp` +
`@modeler/semantics`. **Desktop only.** **Pre-flight:** 4.2 done. This is the
**reverse** direction and needs a project-wide index of SQL refs.

---

- [x] **4.3.1 — Spec (red).** Integration test over a small multi-file project:
      "find references" on a `db` table/column returns its usages **inside SQL
      blocks** across files (plus the existing TTR references if the provider
      already returns those).

- [x] **4.3.2 — SQL reference index.** In `@modeler/semantics`, while resolving
      each `TaggedBlockValue`, record resolved `(symbolQname → [file, span])` into
      a per-project index (rebuilt/invalidated per document version, like the
      existing reference index — search `reference-index` in semantics). Only
      **resolved** refs are indexed (unresolved can't be attributed).

- [x] **4.3.3 — Query the index.** Extend the references provider: when the target
      symbol is a `db` table/column, union the existing TTR references with the SQL
      index entries for that qname. Respect `includeDeclaration`.

- [x] **4.3.4 — Folding-correctness.** Ensure refs that differ only by case/quoting
      (per dialect folding §6.2) are attributed to the same symbol — a `Users` and
      a `users` usage both count. Test across two dialects in one project.

- [x] **4.3.5 — Performance guard.** The index builds during the normal semantics
      pass (no extra parse). Confirm find-refs is O(usages), not a re-scan. Note any
      invalidation edge cases.

- [x] **4.3.6 — Green; lint; commit.**
      ```bash
      pnpm --filter @modeler/semantics --filter @modeler/lsp --filter @modeler/integration-tests test
      pnpm --filter @modeler/semantics --filter @modeler/lsp lint
      ```
