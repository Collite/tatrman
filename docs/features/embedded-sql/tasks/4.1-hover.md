3.4 resolver). **Desktop only.** **Pre-flight:** Phase 3 merged. Study the
existing hover provider in `packages/lsp/src/server.ts`.

- [ ] **4.1.1 — Spec (red).** Integration test (`tests/integration/`): hover over
      `email` in a `"""sql␊SELECT u.email FROM users u␊"""` block returns the TTR
      `db` column's type + description (from the resolved symbol).

- [ ] **4.1.2 — Hit-test SQL position → ref.** Given a hover position inside a
      `TaggedBlockValue`, convert the file position into the SQL-local offset
      (inverse of 2.3.1) and find the `SqlColumnRef`/`SqlTableRef` whose span
      covers it. Reuse the `SqlRefModel` already computed for diagnostics (cache
      per document/version — don't re-parse on hover).

- [ ] **4.1.3 — Resolve + render.** Resolve the ref via the 3.4 resolver to its
      TTR symbol; render Markdown hover: column type, `optional`/key flags,
      description; for a table, its description + column count. Reuse the symbol's
      existing hover formatting if the TTR side already has one.

- [ ] **4.1.4 — Misses degrade quietly.** Hover over an unresolved/unknown
      identifier → no hover (not an error). Hover over a keyword/parameter → no
      symbol hover. Test these.
