# 2.4 — LSP embedded semantic tokens

**Repo:** modeler. **Effort:** ~1 day. **Package:** `@modeler/lsp`.
**Pre-flight:** 2.3 done. Read contracts §7; study the **existing** semantic-token
provider in `packages/lsp/src/server.ts` (search `SemanticTokens`) and the
`semantic-tokens-v1.1.test.ts` harness — extend that, don't replace it.

---

- [ ] **2.4.1 — Extend the token legend.** Add the SQL token types/modifiers from
      contracts §7 (`keyword`, `string`, `number`, `operator`, `comment`,
      `variable`, `parameter`, table→`class`, column→`property`) to the server's
      declared legend. Keep existing TTR token types' indices stable (append).

- [ ] **2.4.2 — Emit embedded SQL tokens (TDD red first).** Add an
      integration test in `tests/integration/` (the canonical PassThrough harness —
      see CLAUDE.md "Cross-package integration tests"): open a `.ttr` doc with a
      `"""sql␊SELECT u.email FROM users u␊"""` block, request semantic tokens,
      assert the `SELECT` keyword and `email`/`users` identifiers come back at the
      **file** positions (via 2.3.1). Red until 2.4.3.

- [ ] **2.4.3 — Wire the provider.** For each `TaggedBlockValue` with a SQL
      `dialect`, call `lexSql(value, dialect)`, classify (2.3.5), map each token to
      a file position (2.3.1), and merge into the document's semantic-token
      response alongside the TTR tokens. Order/encode per the LSP delta spec the
      existing provider already uses.

- [ ] **2.4.4 — Placeholders → `parameter`.** Overlay `masked.placeholders`
      (from `lexSql`) as `parameter` tokens at their file positions, so `{nazev}`
      colours as a parameter (not as the bare identifier the lexer saw). Test one
      `{param}` block asserts the `parameter` token over the full `{nazev}` span.

- [ ] **2.4.5 — Broken-SQL still highlights.** Test: a deliberately malformed
      query (`SELECT FROM WHERE`) still returns keyword tokens for the words it
      lexed (lexer-first tolerates it). No exception escapes the provider.

- [ ] **2.4.6 — Untagged/non-SQL untouched; commit.** Test: an *untagged*
      `"""…"""` description and a `"""transform␊…"""` block emit **no** SQL tokens
      (only SQL dialects are lexed). Run:
      ```bash
      pnpm --filter @modeler/integration-tests test
      pnpm --filter @modeler/lsp test && pnpm --filter @modeler/lsp lint
      ```
      Commit: `Section embedded-sql: LSP embedded SQL semantic tokens`.
