# 3.4 — Resolver + identifier folding + diagnostics

**Repo:** modeler. **Effort:** ~2 days. **Package:** `@modeler/semantics`.
**Pre-flight:** 3.2 (`SqlRefModel`) + 3.3 (`SqlConfig`) done. Read contracts
§5.1 (name resolution), §6.2 (identifier folding), and the existing
`@modeler/semantics` resolver/diagnostics (this is where "unresolved reference"
diagnostics already live — follow that pattern).

---

- [x] **3.4.1 — Diagnostic codes + spec (red).** Define codes:
      `sql-unknown-table`, `sql-unknown-column`, `sql-ambiguous-column`,
      `sql-column-not-on-alias`. Spec (red) against fixtures: unknown table → 1
      diag on the table span; bad column → 1 on the column span; bare column in 2
      joined tables → ambiguous; `a.col` where `col` ∉ alias `a` → not-on-alias.

- [x] **3.4.2 — Identifier folding.** `src/sql/fold.ts` implementing contracts
      §6.2 per dialect (tsql case-insensitive + `[..]`; postgres lower-fold +
      `".."`; duckdb case-insensitive case-preserving). `foldEq(a, b, dialect)`.
      Unit-test `Users` vs `users` equivalence per dialect. **Both** the SQL
      identifier and the TTR symbol name are folded before comparison.

- [x] **3.4.3 — Table resolution (contracts §5.1).** For each `SqlTableRef`: split
      name parts by dialect part-count, fill missing leading parts from
      `defaultsFor(dialect)`, map `(database, schema) → namespace` via the config,
      look up `db.<namespace>.table.<table>` in the TTR symbol table (with folding).
      CTE/derived names resolve to in-query scopes, **not** the symbol table.
      Unresolved → `sql-unknown-table`.

- [x] **3.4.4 — Column resolution + scope.** Qualified `alias.col` → resolve the
      alias to its table (or CTE/derived columns), then the column on that table.
      Bare `col` → search the scope's in-view tables: 0 matches →
      `sql-unknown-column`; ≥2 → `sql-ambiguous-column`; `alias.col` where the
      column isn't on that table → `sql-column-not-on-alias`. Use the 3.2.5 scope
      tree. Make the 3.4.1 specs green.

- [x] **3.4.5 — Wire into the document diagnostics + LSP.** Run SQL resolution per
      `TaggedBlockValue` during the semantics pass; map `SqlRefModel` spans → file
      positions (2.3.1) for each diagnostic. Add an integration test
      (`tests/integration/`) opening a `.ttr` whose SQL references a missing column
      and asserting the diagnostic at the right file range. **Desktop only** — guard
      so the browser worker (no parser) doesn't attempt resolution.

- [x] **3.4.6 — False-positive pass on the corpus; lint; commit.** Resolve the
      S0.2 corpus against a representative `modeler.toml` + `db` schema; review the
      diagnostics for false positives (the Phase 3 DoD). Tune folding/defaults, not
      the tests. Record residual known-FP constructs in `LAZY-PATCH.md`.
      ```bash
      pnpm --filter @modeler/semantics --filter @modeler/integration-tests test
      pnpm --filter @modeler/semantics lint
      ```
