# S0.1 — `antlr-ng` generates tsql + postgresql

**Repo:** modeler (scratch `packages/sql-spike`). **Effort:** ~0.5 day.
**Gate (must pass to continue past Phase 0):** both grammars generate TS that
compiles and instantiates under `antlr4ng`, **and** case-insensitive keyword
matching works — or a concrete fallback is chosen and its cost recorded.

**Pre-flight:**
- Read DESIGN §12.6 (spike) and §12.1.
- Know the toolchain: `packages/parser` generates via `antlr-ng` (script
  `packages/grammar/scripts/generate-typescript-parser.sh`) and runs on `antlr4ng`.

---

- [ ] **S0.1.1 — Scaffold the scratch package.** Create `packages/sql-spike`
      (name `@modeler/sql-spike`, `"private": true`, extends `tsconfig.base.json`,
      depends `antlr4ng` `workspace:*` style like `packages/parser`). This package
      is deleted at the end of Phase 0 — do not wire it into `pnpm -r build`
      defaults beyond what the spike needs.

- [ ] **S0.1.2 — Vendor the grammars pinned.** Download into
      `packages/sql-spike/vendor/tsql/` and `vendor/postgresql/` from
      `antlr/grammars-v4` at a **pinned commit** (record the commit SHA in
      `vendor/PINNED.md`). Files: `TSqlLexer.g4`, `TSqlParser.g4`,
      `PostgreSQLLexer.g4`, `PostgreSQLParser.g4` (+ any imported `.g4` they
      reference). Do **not** edit them yet (lazy patching, DESIGN §12.7).

- [ ] **S0.1.3 — Generate with `antlr-ng`.** Reuse the existing generation script
      as a template; produce TS into `packages/sql-spike/generated/{tsql,postgresql}/`.
      Record the exact command in `vendor/PINNED.md`. **Capture every error/warning
      verbatim** — these are the gate evidence. If a grammar uses
      `options { caseInsensitive = true; }`, note whether `antlr-ng` accepts it.

- [ ] **S0.1.4 — Compile + instantiate smoke test.** Add
      `src/__tests__/generate-smoke.test.ts` (Vitest). For each dialect, lex a
      trivial statement and assert a non-empty token stream:
      ```ts
      import { CharStream, CommonTokenStream } from 'antlr4ng';
      import { TSqlLexer } from '../../generated/tsql/TSqlLexer.js';
      const lx = new TSqlLexer(CharStream.fromString('SELECT 1'));
      const ts = new CommonTokenStream(lx); ts.fill();
      expect(ts.getTokens().length).toBeGreaterThan(1);
      ```
      Run: `pnpm --filter @modeler/sql-spike test`. Must pass for both dialects'
      lexers, and (best-effort) parsers.

- [ ] **S0.1.5 — Case-insensitivity probe.** Lex `select`, `SELECT`, `SeLeCt` and
      assert all yield the `SELECT` keyword token (not `IDENT`). If they don't
      (i.e. `antlr-ng` ignored `caseInsensitive`), implement the fallback per
      DESIGN §12.6: either a case-folding `CharStream` wrapper, or switch to the
      fragment `[Ss][Ee]…` keyword form. **Pick one, implement the smaller, and
      record the choice + effort in the gate notes.**

- [ ] **S0.1.6 — Record the gate verdict.** Create/append
      `packages/sql-spike/spike-report.md` § "S0.1": PASS/FAIL, the pinned SHAs,
      the generation command, any grammar warnings, the case-insensitivity
      approach chosen, and any `antlr-ng` incompatibilities found. **If FAIL**
      (grammars don't generate/compile), stop and trigger the E8 fallback
      re-plan — do not start Phase 2.
