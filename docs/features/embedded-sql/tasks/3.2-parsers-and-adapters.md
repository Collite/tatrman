# 3.2 — Error-tolerant parsers + per-dialect adapters

**Repo:** modeler. **Effort:** ~2 days (the heaviest stage). **Package:**
`@modeler/sql`. **Pre-flight:** 3.1 red specs in place. Read DESIGN §12.3/§12.4
and SPIKE S0.2 (default error strategy, masking). This is the **best-effort**
parser layer — cover the common constructs; let exotic syntax degrade.

---

- [ ] **3.2.1 — Parser service (error-tolerant).** `src/parser-service.ts`:
      `parseSql(value, dialect): { tree, parseErrors }`. Use `maskPlaceholders`
      first (same as the lexer path), then the dialect parser with the **default
      error strategy** (NOT `BailErrorStrategy` — DESIGN §12.3) and an error
      listener collecting `{ message, span }`. Entry rule: tsql `tsql_file()`,
      postgres `root()` (confirm from generated parser). Never throw.

- [ ] **3.2.2 — Adapter framework.** `src/adapters/index.ts`:
      `extract(tree, dialect)` dispatches to `TsqlAdapter` / `PostgresAdapter`
      (DuckDB → Postgres adapter). Each is an antlr4ng tree visitor/listener that
      emits the **shared** `SqlRefModel` (DESIGN §12.4 — normalise the extraction,
      not the trees). Walk defensively: skip `ErrorNode`s, push to `parseErrors`.

- [ ] **3.2.3 — T-SQL adapter.** Implement table refs (`FROM`/`JOIN`, `[bracket]`
      and 1–4-part names → `name: string[]`), aliases, CTEs (`WITH`), derived
      tables, column refs (qualified `a.col` + bare `col`), params (`@p` + the
      masked `{param}` spans threaded in from the lexer pass). Make
      `extract-tsql.test.ts` green. Use the generated `TSqlParser` context classes
      (inspect `src/generated/tsql/TSqlParser.ts` for rule/context names).

- [ ] **3.2.4 — PostgreSQL adapter.** Same surface for Postgres syntax
      (`"quoted"` ids, `schema.table`, `$1`/`:name` params, dollar-quoted strings
      already handled by the lexer). Make `extract-postgres.test.ts` green. DuckDB
      reuses this adapter (add a thin alias; patch later only if a real DuckDB
      construct fails — DESIGN §12.7).

- [ ] **3.2.5 — Scope builder.** Shared `src/scope.ts`: from the adapter output,
      build the `SqlScope` tree (CTE names + table aliases per query level, nested
      for subqueries). Make the 3.1.5 scope spec green. This is what lets the
      resolver decide bare-column ambiguity.

- [ ] **3.2.6 — Corpus regression; lint; commit.** Run `extract` over the S0.2
      corpus; assert no throws and that `parseErrors` is empty for the
      cleanly-parsing files (it was 100% in the spike). Record any new failing
      construct in `packages/sql/LAZY-PATCH.md` (DESIGN §12.7 backlog — starts
      empty).
      ```bash
      pnpm --filter @modeler/sql test && pnpm --filter @modeler/sql lint
      ```
      Commit: `Section embedded-sql: error-tolerant parsers + dialect adapters`.
