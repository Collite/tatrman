# 2.3 — SQL lexer service + dialect selection + source map

**Repo:** modeler. **Effort:** ~1 day. **Package:** `@modeler/sql` (service) +
`@modeler/parser` (source-map helper). **Pre-flight:** 2.1, 2.2 done. Read
DESIGN §8 (source map), contracts §3 (tag registry) and §3a (mask).

---

- [ ] **2.3.1 — Source-map helper (TDD).** In `@modeler/parser` (next to the
      `TaggedBlockValue` walker), add `sqlPosToFile(tok, block)`:
      ```
      fileLine = block.valueSource.startLine + tok.line - 1
      fileCol  = (tok.line === 1 ? block.valueSource.startColumn : 0)
                 + block.indentWidth + tok.column
      ```
      Spec it against the SPIKE S0.2 span samples (e.g. `email` at line 1 col 9 in
      `SELECT u.email FROM users u` inside a known block → exact file position).
      Cover a second-line token (uniform `indentWidth` added, column from 0).

- [ ] **2.3.2 — Lexer service (TDD red first).** `src/lexer-service.ts` in
      `@modeler/sql`:
      ```ts
      export interface SqlToken { type: number; typeName: string; span: Span; }
      export function lexSql(value: string, dialect: SqlDialect):
        { tokens: SqlToken[]; masked: MaskResult };
      ```
      Spec: lexing a `{param}` query yields keyword/identifier tokens and **zero**
      error tokens (the mask did its job). Red until 2.3.3.

- [ ] **2.3.3 — Implement dialect selection + lex.** `lexSql`:
      1. `const m = maskPlaceholders(value)`.
      2. pick the lexer class by `dialect` (`tsql` → `TSqlLexer`, `postgres`/`duckdb`
         → `PostgreSQLLexer`) — DuckDB reuses the Postgres lexer (E11).
      3. `CharStream.fromString(m.masked)` → lexer → `CommonTokenStream`,
         `tokens.fill()`, map each non-EOF token to `SqlToken` with a `Span`
         (`tok.start`, `tok.stop`, `tok.line`, `tok.column`).
      Use the antlr4ng pattern from INDEX. Return `{ tokens, masked: m }`.

- [ ] **2.3.4 — Dialect resolution from the tag.** Add `resolveDialect(block,
      sqlConfig?)`: `block.dialect ?? sqlConfig.defaultDialect`. For Phase 2 the
      config may be absent → fall back to a hard default (`tsql`) with a TODO
      pointing at Phase 3's `modeler.toml` loader. Unit-test both paths.

- [ ] **2.3.5 — Token-type → semantic-token-type map.** `src/token-classify.ts`:
      map ANTLR token types to the LSP legend names (contracts §7): keywords →
      `keyword`, string literals → `string`, numerics → `number`, comments →
      `comment`, operators/punct → `operator`, T-SQL `@var`/`#temp` → `variable`.
      Drive it off token-type ranges / names, not hard-coded integers (regen-safe).
      Unit-test a representative line per dialect.

- [ ] **2.3.6 — Lex-coverage regression; lint; commit.** Port the spike's
      coverage harness as a test: every S0.2 corpus file lexes with zero error
      tokens (tsql via mask, postgres natively). Then:
      ```bash
      pnpm --filter @modeler/sql --filter @modeler/parser test
      pnpm --filter @modeler/sql --filter @modeler/parser lint
      ```
      Commit: `Section embedded-sql: SQL lexer service + source map`.
