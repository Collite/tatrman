# 1.1.F — Migration CLI

**Goal:** ship `modeler migrate-to-packages <project-root>`, a CLI that converts a v1 project (no packages, no imports, project-wide `.ttrl`) into a v1.1 project (package declarations, import statements, per-graph `.ttrg` files).

**Reads:** [contracts §10 (migration CLI surface)](../../design/v1-1-contracts.md#10-migration-cli-surface), [design doc §9 (migration tooling)](../../design/v1.1-packages-and-graphs.md#9-migration-tooling), existing `samples/v1-mini/`, `samples/v1-metadata/`.
**Blocked by:** 1.1.B.4 (uses resolver to scan cross-references).
**Blocks:** G (migrates the samples using this CLI).
**Estimated time:** 3–5 days.

## Tests-first

- [ ] `packages/migrate/src/__tests__/infer-package.test.ts` — unit. Cases:
  - File at `<root>/foo/bar/baz.ttr` infers `foo.bar`.
  - File at `<root>/baz.ttr` infers `''` (default package).
  - File at `<root>/foo/baz.ttr` infers `foo`.
- [ ] `packages/migrate/src/__tests__/insert-package.test.ts` — unit. Cases:
  - File with no `package` line: insert `package X.Y\n` at the very top.
  - File with an existing `package X.Y` line: no-op (idempotent).
  - File with leading comments: insert `package` after the leading comment block.
- [ ] `packages/migrate/src/__tests__/insert-imports.test.ts` — unit. Cases:
  - File in package `A` referencing `<pkg-B-qname>` produces one `import B.<schema>.<ns>.<defName>` (named form) when only one symbol is referenced, or `import B.*` (wildcard) when ≥3 symbols from package B are referenced. Threshold (3) is configurable via `--wildcard-threshold`.
  - Idempotent: re-running the CLI doesn't re-add existing imports.
- [ ] `packages/migrate/src/__tests__/ttrl-to-ttrg.test.ts` — unit. Cases:
  - `.ttrl` with `viewports.db` and `viewports.er` produces two `.ttrg` files: `graphs/_all_db.ttrg` and `graphs/_all_er.ttrg`. Each contains every db/er object respectively, with the layout positions preserved.
- [ ] `tests/integration/src/migration.test.ts` — end-to-end. Runs the CLI on a copy of `samples/v1-mini/` in a temp dir and asserts:
  - Every `.ttr` parses cleanly under v1.1 grammar.
  - Every cross-reference resolves with no `ttr/unresolved-reference` or `ttr/unimported-reference`.
  - `_all_er.ttrg` opens via `client.getGraph` with `missingObjects === []`.

## Implementation tasks

- [ ] **F.1 — Create `packages/migrate/`.** Skeleton: `package.json`, `tsconfig.json` extending `tsconfig.base.json`, `src/index.ts` (programmatic API), `src/cli.ts` (CLI entry, bin: `modeler-migrate`). Add workspace dep to `pnpm-workspace.yaml`.
- [ ] **F.2 — Implement `inferPackage(filePath, projectRoot): string`.** Pure function. Per the path-relative rule in [design doc §3.2](../../design/v1.1-packages-and-graphs.md#32-package--directory).
- [ ] **F.3 — Implement `insertPackageDecl(fileContent, packageName): string`.** Pure function. Handles leading comments. Idempotent. Returns unchanged content if `package` already present.
- [ ] **F.4 — Implement `scanCrossReferences(fileContent, fromPackage, projectSymbols): ImportSpec[]`.** Parses the file's AST, walks every reference, returns the deduplicated list of imports needed. Wildcard-vs-named decision per the threshold flag.
- [ ] **F.5 — Implement `insertImports(fileContent, imports): string`.** Inserts the import block after the `package` line, before `schema`, in alphabetical order. Idempotent.
- [ ] **F.6 — Implement `convertTtrlToTtrg(ttrlPath, projectRoot, projectSymbols): WriteOp[]`.** Reads `.modeler/layout.ttrl`, produces one `.ttrg` per schema present in the layout. Filename pattern: `_all_<schema>.ttrg` under `<projectRoot>/graphs/`. Each `.ttrg` lists every object of that schema in the project's symbol table (computed at migration time) and preserves the layout positions. Hand-document at the top of each generated file: `// Generated by modeler migrate-to-packages; rename/split as desired.`
- [ ] **F.7 — Wire `cli.ts`.** Flags: `--dry-run` (no writes), `--commit-ttrl-removal` (delete `.ttrl` after success), `--wildcard-threshold N` (default 3), `--verbose`. Exit codes per [contracts §10](../../design/v1-1-contracts.md#10-migration-cli-surface). On `--dry-run`, prints the `MigrateReport` shape; on real run, prints a summary + writes the report to `<root>/.modeler/migrate-report.json`.
- [ ] **F.8 — Handle ambiguous references.** When `scanCrossReferences` can't decide which package a bare reference belongs to (two packages export the same name), include the entry in `MigrateReport.ambiguousReferences` and skip auto-import for that ref. Exit code `1`. The user resolves manually then re-runs.
