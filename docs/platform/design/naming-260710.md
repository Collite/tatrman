# Tatrman Naming — Final Ledger & Rename Task List (2026-07-10)

> Ratified in the ecosystem strategy session 2026-07-10. Supersedes the platform design's
> J-register rule ("platform-native services get Slavic names, transplants keep Greek").
> **New rule: functions name contracts, modules, and repos (`ttr-<function>`,
> `org.tatrman.<function>.v1`); personas name only the kept few — and personas NEVER
> appear in proto packages, Maven coordinates, or published artifact names.**
> Execute BEFORE the publish gates (ttr-metadata GitHub Packages, ttr-translator/ttr-plan-proto
> transfer): rename first, publish once.

## 1. Brand architecture (public vocabulary — four brands + one product suffix)

| Brand | What it names |
|---|---|
| **Tatrman** | The standard & toolchain: languages (TTR-M/TTR-P/TTR-B), formats, compiler, CLI, IDE, Designer ("Tatrman Designer") |
| **Tatrman Server** | The open spine as a deployable product (metadata + governed query runtime; one name, many services — cf. "SQL Server") |
| **Tatrman Platform** | The commercial operate tier (parked satellite; name pre-exists) |
| **Kantheon** | The intelligence suite (agents). Persona names are legal and encouraged INSIDE Kantheon |
