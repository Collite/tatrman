`ai-models` repo at `~/Dev/ai-models`. The `model-ttr/` folder holds the **actual** TTR model
content used by ai-platform (the metadata service reads it via fabric-infra
`METADATA_GIT_SUBDIR=model-ttr`, `METADATA_GIT_KIND=ttr`).

Structure: package dirs (artikl, ucetnictvi, prodeje, …) each with `db.ttr`/`er.ttr`/`cnc.ttr`/etc
(~41 `.ttr` files); `domains/*.ttrd` (2 files: accounting, obchod) = subject-area scope for the
Golem agent registry; `model-ttr/resolved-packages.json` generated by the **Modeler CLI**
(`modeler resolve-packages model-ttr --out …`, driven by repo `justfile`); `modeler.toml` declares
schemas `db`/`er`/`cnc` only (**no `schema map`**). ai-models' own tools (`tools/*.py`) scan only
`agents/**/*.yaml` + consume resolved-packages.json — they don't read `.ttr` files.

Phase 0 Stage E migrates this content to grammar 3.0 (`.ttr`→`.ttrm`, `.ttrd` domain → `def area`),
regenerates resolved-packages.json, and verifies the metadata service loads it. See
[[md-model-design]], [[ai-platform-repo]].
