Designing a new TTR feature: the multidimensional (ROLAP-first, MOLAP-later) model + its binding to physical/ER, and (later) an operations DSL. Brief + old RAE examples live in `docs/features/md/`.

Firm decisions (2026-06-23 brainstorm):
- **`domain`** = the NEW MD value-set concept (typed value space with restriction rules). One unified concept with several restriction forms (enum/member-set, range like 1..100, pattern, etc. — extensible).
- The existing 2.3 `.ttrd`/`domain` (subject-area grouping) is being **renamed to `area` / `.ttra`** to free up `domain`. Low real-world usage, acceptable breaking change (coordinate with ai-platform agent registry).
- Cross-model "mapping" is being **renamed to `binding`** (so `schema map`/`er2db_*` → binding family) to free the word "map/mapping" for the MD primitive.
- **`attribute`** is kept in BOTH ER and MD schemas (similar meaning); use one permissive grammar body + per-schema semantic validation (repo's existing "parser stays mechanical" pattern).
- **Maps operate on Domains**, not attributes. "map this attribute to that" is sugar over mapping the underlying domains.
- **A map is a function** `A×…→X` ⇒ N:1 or 1:1 in the forward/roll-up direction ("avoid M:N" = "be a function"). Two backings: **table-backed** (extensional, case list in a table; the case-table is a binding concern) and **calc-backed** (intensional/function). Inverse (drill-down/writeback) isn't a function → needs a strategy (e.g. top(1) winner), deferred.
- **Calc maps (v1): built-in library, no general expression language.** Ship a fixed catalog of parameterized, named standard maps (truncToDay, monthOfDate, quarterOfMonth, yearOfDate…) referenced by name in the map def (`calc: quarterOfMonth`). Time is modeled via these. Custom calc logic (e.g. plate→county) is OUT of v1 — that's when a scalar-expression tier returns.
- Built-in map functions are **abstract/dialect-agnostic**; the binding/translator (ai-platform runtime) lowers them per SQL dialect (Teradata etc.). ⇒ the catalog is a **cross-repo contract**: modeler ships signatures (for validation, like the stock CNC vocab), ai-platform ships lowerings. Needs a home + sync story like TTR.g4.

More settled decisions:
- **Domain `kind: calc | bound`** (optional, only for member-set domains; scalar domains like Money have no kind). `bound` = members come from physical model (binding supplies source); `calc` = members generated by rule (Month=1..12, calendar). Authoritative when present (enables "kind:bound but no source → error"). Parallels map calc/table backing (domain kind = how members exist; map kind = how the function is given).
- **Dimension**: allows short **inline** attribute defs (like ER entity). Declares **exactly one `key`** attribute (member identity). `nameAttribute`/`codeAttribute` dropped (name/code display handled differently later).
- **Leaves are emergent, not declared**: an attribute is a leaf if no **N:1** map targets it. Grain = partial order from N:1 maps. **1:1 maps are same-grain aliases** (co-leaves, e.g. code+id), don't demote leaf-ness. Two attributes can share a domain; "map attribute A to B" sugar resolves to the map between their domains (surfaced to authors).
- **Hierarchy** = a named ordered path through the N:1 lattice. Connecting map between consecutive levels is **inferred by default**; **`via:`** overrides; ambiguity without `via:` is an **error**.

- **Hierarchy levels written leaf→root** (`[Day, Month, Quarter, Year]`).
- **Measure additivity**: every measure carries `class: additive | semiAdditive | nonAdditive` (default additive). additive → single agg. semiAdditive → default agg + per-dimension overrides + validity marker, e.g. `aggregation: { default: sum, time: latestValid }`, `validBy: <attr>` (covers latest-valid). nonAdditive → in v1 just *marked* (tooling refuses blind sum); the num/den recompute formula waits for the expression tier (v1.1). Measures are standalone defs, also allowed inline in a cubelet.

Proposed 7 logical objects (binding-free): domain, dimension, attribute, map (N domains→M domains, table- or calc-backed, with cardinality), hierarchy (ordered attribute levels, realized by maps), measure (domain + default aggregation), cubelet (grain = attribute set + measures). Bindings: md→db (shape wide/long, journaling overwrite/invalidate/diff, multi-source) and md→er.

Phase 0 (legacy renames, before MD work) — grammar major bump to **3.0**, task list at `docs/features/md/plan/phase-0-legacy-renames/`: (A) `schema map`→`schema binding` (er2db_* defs unchanged); (B) drop `domain` block + `.ttrd` file kind, replace with **`def area`** (a normal def — user agreed area is model content, no file kind; ai-platform agent registry must discover area defs instead of .ttrd files); (C) `.ttr`→`.ttrm` ("Tatrman Model"; `.ttrg` unchanged); (D) version bump + sync to ai-platform/Kotlin conformance + migration helper + dead `.ttrl` cleanup. `attribute` keyword kept in both er & md (no rename).

Session approach: model layer first; ops DSL paradigm deferred (exploring tradeoffs later). See [[modeler_repo_path]] and [[ai_platform_repo]].
